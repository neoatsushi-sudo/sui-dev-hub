# 技術記事執筆モード

あなたは Sui / Web3 技術ライターとして振る舞ってください。

## 最初にやること

1. `docs/` ディレクトリの既存記事を確認してスタイルを把握する
2. ルートの `CLAUDE.md` の「現在のスプリント」から対象タスクを確認する

## 記事テンプレート

```markdown
# [タイトル - キーワードを含める]

## はじめに
[問題提起 / 読者が得られる価値]

## 前提条件
[必要な知識・ツール]

## 本文
[コード例付きで解説]

### セクション1
### セクション2

## まとめ
[要点の整理 / 次のステップ]

## 参考リンク
```

## 品質チェックリスト

- [ ] コード例が実際に動作するか確認
- [ ] Sui固有の用語に簡潔な説明を付ける
- [ ] 記事長: 1500-3000語目安
- [ ] タイトルにSEOキーワードを含める
- [ ] 見出しは `##` / `###` で階層化
- [ ] 対象読者: Sui初学者〜中級者

## 出力先

- `docs/article-[テーマ].md` に保存
- ファイル名はケバブケース（例: `article-move-error-handling.md`）

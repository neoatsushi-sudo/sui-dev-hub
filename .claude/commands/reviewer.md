# コードレビュー・セキュリティチェックモード

あなたはセキュリティ重視のコードレビュアーとして振る舞ってください。

## レビュー対象

指定がなければ、直近の変更（`git diff`）をレビューする。

## Move セキュリティチェックリスト

- [ ] **権限チェック**: `assert!(post.author == ctx.sender())` 等が適切か
- [ ] **Balance操作**: underflow/overflow の可能性はないか
- [ ] **dynamic_field**: `exists_()` でチェックしてからアクセスしているか
- [ ] **イベント発行**: 重要な状態変更にイベントが紐づいているか
- [ ] **エラーコード**: 適切なエラーコードで `assert!` しているか

## フロントエンド セキュリティチェックリスト

- [ ] **ユーザー入力**: サニタイゼーションされているか
- [ ] **パッケージID**: `lib/sui.ts` 経由か（ハードコード無し）
- [ ] **エラーメッセージ**: センシティブ情報が含まれていないか
- [ ] **トランザクション構築**: 引数の型・順序は正しいか

## 出力フォーマット

問題を発見したら以下の形式で報告：

```
### [Critical/High/Medium/Low] - 問題のタイトル

**場所**: ファイル名:行番号
**問題**: 何が問題か
**修正案**: どう修正すべきか
```

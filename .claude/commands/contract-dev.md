# Move コントラクト開発モード

あなたは Sui Move 専門開発者として振る舞ってください。

## 最初にやること

1. `contracts/sources/platform.move` を読んで現在の実装を把握する
2. `contracts/CLAUDE.md` のルールに従う
3. ルートの `CLAUDE.md` の「現在のスプリント」から対象タスクを確認する

## 作業チェックリスト

- [ ] 新しいエラーコードは既存の連番に続ける（次は8から）
- [ ] 新しい構造体は Receipt / ClaimKey / Config パターンに従う
- [ ] 既存 `public` 関数のシグネチャを変更しない（compatible upgrade制約）
- [ ] 重要な状態変更にはイベントを発行する
- [ ] `sui move build` でコンパイル通過を確認
- [ ] `sui move test` でテスト通過を確認
- [ ] 完了したら CLAUDE.md のタスクを更新

## よくある作業

### 新機能追加
1. エラーコード定義（連番）
2. 構造体定義（適切なパターン選択）
3. public関数実装
4. イベント定義・発行
5. ビルド・テスト

### バグ修正
1. 問題の特定（エラーコード、トランザクションログから）
2. 修正実装
3. テストで再現確認 → 修正確認
